# 力量举术语规范

本文档整理力量举运动的标准术语，用于指导项目中的命名规范。

## 📚 力量举标准术语

### 核心训练术语

| 英文术语 | 中文翻译 | 说明 | 示例 |
|---------|---------|------|------|
| **Training Session** | 训练课 | 一次完整的训练 | "今天的训练课" |
| **Workout** | 训练 | 同 Training Session | "今天的训练" |
| **Exercise** | 动作/练习 | 具体的训练动作 | "卧推"、"深蹲"、"硬拉" |
| **Exercise Block** | 动作块 | 训练记录中的动作块（包含该动作的所有组） | "今天训练中的卧推块" |
| **Exercise Entry** | 动作条目 | 同 Exercise Block（标准术语，但语义可能混淆） | "今天训练中的卧推记录" |
| **Exercise Record** | 动作记录 | 同 Exercise Block（标准术语，但语义可能混淆） | "动作记录" |
| **Set** | 组 | 连续完成一定次数动作的集合 | "3组卧推" |
| **Rep** / **Repetition** | 次数 | 每次完成一个完整动作 | "10次" |
| **1RM** | 一次最大重量 | 能够完成一次的最大重量 | "卧推1RM: 100kg" |
| **Volume** | 训练量 | 总重量（组数 × 次数 × 重量） | "训练量: 3000kg" |
| **RPE** | 主观疲劳度 | Rate of Perceived Exertion | "RPE 8" |

## 📋 当前项目命名分析

### 当前命名

| 域名 | 当前命名 | 实际含义 | 问题 |
|-----|---------|---------|------|
| 训练课 | `workout` | 一次训练 | ✅ 符合标准术语 |
| 动作模板 | `exercise` | 训练动作 | ✅ 符合标准术语 |
| 动作条目 | `workoutset` | 训练中的动作记录 | ❌ 不是标准术语 |
| 组 | `set` | 一组训练 | ✅ 符合标准术语 |
| 身体部位 | `bodypart` | 身体部位 | ⚠️ 可优化为 `body-part` 或 `muscle-group` |

### 命名问题

1. **`workoutset`** - 不是力量举标准术语
   - 当前语义：容易理解为"训练集合"
   - 实际含义：训练记录中的动作块（包含该动作的所有组）
   - 推荐术语：**Exercise Block**（语义清晰，避免与单个 Set 混淆）

2. **`bodypart`** - 可优化
   - 当前：`bodypart`（一个词）
   - 建议：`body-part` 或 `muscle-group`（更专业）

## 💡 基于力量举术语的命名建议

### 方案 A：使用 Exercise Block（推荐）

| 当前 | 建议 | 理由 |
|---|---|---|
| `workout` | `workout` | ✅ 保持，标准术语 |
| `exercise` | `exercise` | ✅ 保持，标准术语 |
| `workoutset` | `exercise-block` | ✅ 语义清晰，表示"动作块"（包含所有组） |
| `set` | `set` | ✅ 保持，标准术语 |
| `bodypart` | `body-part` | ✅ 优化格式，或改为 `muscle-group` |

**优点：**
- 语义清晰：Block 明确表示"一个完整的块"，包含该动作的所有 Sets
- 避免歧义：不会与单个 Set 混淆
- 符合直觉：在 UI 中，一个 Block 就是一个可编辑的功能块
- 易于理解：符合组件化思维

### 方案 B：使用 Exercise Entry（标准术语，但语义可能混淆）

| 当前 | 建议 | 理由 |
|---|---|---|
| `workoutset` | `exercise-entry` | ✅ 使用力量举标准术语 |
| 其他 | 同方案 A | - |

**优点：**
- 符合力量举标准术语规范
- 专业性强

**缺点：**
- "Entry" 可能让人误解为单个 Set，而不是所有 Sets 的集合

### 方案 C：简化命名

| 当前 | 建议 | 理由 |
|---|---|---|
| `workoutset` | `exercise-log` | ✅ 简洁，表示"动作日志" |
| 其他 | 同方案 A | - |

**优点：**
- 简洁明了
- 虽然不是标准术语，但语义清晰

## 🎯 推荐方案

### 首选：方案 A - Exercise Block

**理由：**
1. ✅ 语义清晰：Block 明确表示"一个完整的块"，包含该动作的所有 Sets
2. ✅ 避免歧义：不会与单个 Set 混淆（Entry 可能让人误解为单个条目）
3. ✅ 符合直觉：在 UI 中，一个 Block 就是一个可编辑的功能块
4. ✅ 易于理解：符合组件化思维

**注意：** 虽然 Exercise Block 不是标准力量举术语，但语义更清晰，适合项目内部使用。
