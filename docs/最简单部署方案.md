# 最简单部署方案

## 原则

**不要任何健康检查，直接停旧容器，启动新容器，让容器自己跑。**

## 部署命令

```bash
cd /path/to/deploy/frontend/deploy

# 停止旧容器并启动新容器
docker compose down && docker compose up -d
```

**就这一行命令，完成！**

## 配置说明

- ✅ **移除了所有健康检查**：不检查应用是否启动成功
- ✅ **保留自动重启**：`restart: unless-stopped`，容器崩溃会自动重启
- ✅ **保留依赖等待**：确保 PostgreSQL 健康后才启动应用

## 验证（可选）

如果想看看容器是否在运行：

```bash
# 查看容器状态
docker compose ps

# 查看日志
docker compose logs gymapp
```

## 如果容器启动失败

查看日志找原因：

```bash
docker compose logs gymapp --tail 50
```

## 优势

- ✅ **极简**：只有一个命令
- ✅ **无干扰**：没有健康检查干扰
- ✅ **可靠**：Docker 自动重启机制
- ✅ **快速**：立即完成，不等待
